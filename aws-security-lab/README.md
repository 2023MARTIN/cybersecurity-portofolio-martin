# AWS Security Lab

## Project Overview

This project demonstrates how to design and secure a basic AWS environment using core security services and best practices.

## Objectives

- Create a secure AWS lab environment
- Configure IAM users, groups, and least-privilege access
- Enable CloudTrail for logging
- Enable GuardDuty for threat detection
- Configure Security Groups for traffic restriction
- Document findings and security improvements

## Lab Components

- VPC
- Subnet
- EC2 instance
- IAM users and roles
- CloudTrail
- GuardDuty
- Security Groups

## Project Structure

- `architecture/` → architecture diagrams
- `screenshots/` → screenshots of AWS configuration
- `configs/` → policy examples, notes, config details
- `reports/` → final security report

## Expected Outcomes

By the end of this lab, the environment should:
- follow least-privilege principles
- generate audit logs
- detect suspicious behavior
- demonstrate secure cloud design basics
